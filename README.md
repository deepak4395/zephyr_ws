# Zephyr Multi-Target Workspace (`zephyr_ws`)

This repository contains a multi-board Zephyr RTOS workspace with:
- ESP32 LED blink project
- Raspberry Pi Pico LED blink project
- Two native POSIX simulation projects
- Setup automation scripts for reproducible builds

The workspace is designed to let you switch targets quickly while keeping one shared Zephyr workspace.

## 1. Project Structure and Directory Purpose

Current top-level layout:

```text
zephyr_ws/
├── .git/                       # Git metadata
├── .gitignore                  # Ignore rules
├── .west/                      # west workspace metadata (created by west init)
├── README.md                   # This document
├── esp32_led_blink/            # ESP32 Zephyr app + manifest + build outputs
├── pico_led_blink/             # Raspberry Pi Pico Zephyr app + manifest + build outputs
├── native_project/             # Native POSIX Zephyr app + manifest + build outputs
├── native_project_2/           # Second native POSIX Zephyr app + manifest + build outputs
├── scripts/                    # Setup and toolchain helper scripts
├── venv/                       # Python virtual environment used by west and Zephyr tooling
├── zephyr/                     # Zephyr RTOS source tree (fetched by west)
├── modules/                    # Zephyr modules fetched/managed by west
└── zephyr-sdk-0.16.8/          # Zephyr SDK toolchains and host tools
```

### Directory usage details

- `.west/`
	- Created by `west init`.
	- Stores workspace metadata and manifest state.
	- Required for `west update`, `west build`, and Zephyr workspace management.

- `esp32_led_blink/`
	- `west.yml`: project manifest for Zephyr + `hal_espressif`.
	- `app/`: actual ESP32 application source (`CMakeLists.txt`, `prj.conf`, `src/main.c`).
	- `build_esp32_devkitc_wroom/`: generated build output for `esp32_devkitc_wroom`.

- `pico_led_blink/`
	- `west.yml`: project manifest for Zephyr + `hal_rpi_pico` + `cmsis`.
	- `app/`: Pico application source.
	- `build_rpi_pico/`: generated build output for `rpi_pico`.

- `native_project/` and `native_project_2/`
	- Each has its own `west.yml` + application source.
	- `build_native_posix_64/`: generated host-simulation build output.
	- `build_native/`: additional/legacy build directory that may exist from previous builds.

- `scripts/`
	- `setup.sh`: idempotent project setup + dependency install + build automation.
	- `install_esp32_toolchain.sh`: SDK and ESP32 flashing tool installation helper.

- `venv/`
	- Python virtual environment used by `west` and Zephyr Python dependencies.
	- Created if absent by `scripts/setup.sh`.

- `zephyr/`
	- Zephyr RTOS main source tree fetched by `west update` based on active manifest.
	- Upstream-managed; avoid random edits unless intentionally patching Zephyr.

- `modules/`
	- HAL/module repositories pulled by active manifests.
	- Includes entries such as `hal/espressif`, `hal/rpi_pico`, and `hal/cmsis`.

- `zephyr-sdk-0.16.8/`
	- Zephyr SDK toolchains and host tools.
	- Used for hardware targets (ESP32, Pico).

## 2. What Setup Creates or Updates

Running setup/install scripts creates or updates these workspace artifacts:

1. `scripts/install_esp32_toolchain.sh`
	 - Ensures `zephyr-sdk-0.16.8/` exists.
	 - Installs `esptool.py`.
	 - Creates `esp32_env.sh` in repo root.
	 - Note: `esp32_env.sh` is generated by installer and may be absent before installer runs.

2. `scripts/setup.sh <project_name>`
	 - Creates `venv/` if missing.
	 - Installs/updates `west` in venv.
	 - Initializes `.west/` if missing.
	 - Sets active manifest with:

	 ```sh
	 west config manifest.path <project_name>
	 ```

	 - Runs `west update` to sync Zephyr and module repos.
	 - Installs Zephyr Python requirements.
	 - Builds selected project into project-specific build directory:
		 - `esp32_led_blink/build_esp32_devkitc_wroom/`
		 - `pico_led_blink/build_rpi_pico/`
		 - `<native_project>/build_native_posix_64/`

Important: Switching between projects changes active manifest path, so `west` commands operate on the currently selected project manifest.

## 3. `esp32_led_blink` (ESP32 DevKitC WROOM)

### Purpose
Demonstrates basic GPIO output on ESP32 with Zephyr by toggling LED pin 13 every 1 second.

### Key files
- `esp32_led_blink/west.yml`
- `esp32_led_blink/app/src/main.c`
- `esp32_led_blink/app/prj.conf`

### Prerequisites
- Zephyr SDK available at `zephyr-sdk-0.16.8/`
- Python venv + `west`
- `esptool.py` for flashing
- ESP32 board (`esp32_devkitc_wroom`) connected via USB

### Build

```sh
sh scripts/setup.sh esp32_led_blink
```

Rebuild only:

```sh
source venv/bin/activate
west build -b esp32_devkitc_wroom esp32_led_blink/app -d esp32_led_blink/build_esp32_devkitc_wroom
```

### Flash

```sh
esptool.py --chip esp32 --port /dev/ttyUSB0 --baud 460800 write_flash \
	0x1000 esp32_led_blink/build_esp32_devkitc_wroom/zephyr/bootloader-esp32.bin \
	0x8000 esp32_led_blink/build_esp32_devkitc_wroom/zephyr/partitions.bin \
	0x10000 esp32_led_blink/build_esp32_devkitc_wroom/zephyr/zephyr.bin
```

### Serial monitor

```sh
picocom /dev/ttyUSB0 -b 115200
```

### Expected runtime log

```text
ESP32 LED Blink starting...
LED configured on pin 13. Starting blink...
LED ON
LED OFF
...
```

### Main outputs
- `esp32_led_blink/build_esp32_devkitc_wroom/zephyr/zephyr.bin`
- `esp32_led_blink/build_esp32_devkitc_wroom/zephyr/bootloader-esp32.bin`
- `esp32_led_blink/build_esp32_devkitc_wroom/zephyr/partitions.bin`

## 4. `pico_led_blink` (Raspberry Pi Pico)

### Purpose
Demonstrates LED toggle using device-tree alias `led0` and Zephyr GPIO DT APIs.

### Key files
- `pico_led_blink/west.yml`
- `pico_led_blink/app/src/main.c`
- `pico_led_blink/app/prj.conf`

### Prerequisites
- Zephyr SDK at `zephyr-sdk-0.16.8/`
- Raspberry Pi Pico board
- `west` tooling in venv

### Build

```sh
sh scripts/setup.sh pico_led_blink
```

Rebuild only:

```sh
source venv/bin/activate
west build -b rpi_pico pico_led_blink/app -d pico_led_blink/build_rpi_pico
```

### Flash options

Option A (UF2 drag-and-drop, recommended):

1. Hold BOOTSEL and connect Pico.
2. Copy UF2 file to mounted `RPI-RP2` drive.

```sh
cp pico_led_blink/build_rpi_pico/zephyr/zephyr.uf2 /media/$USER/RPI-RP2/
```

Option B (runner-based):

```sh
west flash -d pico_led_blink/build_rpi_pico
```

### Expected runtime behavior
- Pico onboard LED toggles every 500 ms.
- Startup log includes:

```text
Raspberry Pi Pico LED blink started
```

### Main outputs
- `pico_led_blink/build_rpi_pico/zephyr/zephyr.uf2`
- `pico_led_blink/build_rpi_pico/zephyr/zephyr.elf`

## 5. `native_project` (Native POSIX Simulation)

### Purpose
Runs Zephyr as host simulation (`native_posix_64`) and prints a hello message.

### Key files
- `native_project/west.yml`
- `native_project/app/src/main.c`
- `native_project/app/prj.conf`

### Build

```sh
sh scripts/setup.sh native_project
```

Manual build command used by script:

```sh
west build -b native_posix_64 native_project/app -d native_project/build_native_posix_64 -p always -- -DZEPHYR_TOOLCHAIN_VARIANT=host
```

### Run

```sh
./native_project/build_native_posix_64/zephyr/zephyr.exe
```

### Expected output

```text
Hello from native_project (Zephyr v3.7.0, native_posix_64)
```

## 7. Setup Scripts Reference

### `scripts/setup.sh`
- Idempotent setup and build script.
- Supports interactive project selection or direct argument.
- Detects board from project name:
	- `esp32*` -> `esp32_devkitc_wroom`
	- `pico*` -> `rpi_pico`
	- `native*` -> `native_posix_64`
- For native target, adds `-DZEPHYR_TOOLCHAIN_VARIANT=host`.
- For hardware targets, exports SDK environment from `zephyr-sdk-0.16.8/`.

### `scripts/install_esp32_toolchain.sh`
- Installs/validates Zephyr SDK.
- Installs `esptool.py`.
- Creates helper `esp32_env.sh`.

### `esp32_env.sh` (generated)
- Generated by installer script.
- Convenience environment setup for ESP32 toolchain variables.
- May not exist until installer script has been run at least once.

## 8. Vendored/Managed Content Boundaries

- Upstream-managed (fetched/synced by `west`):
	- `zephyr/`
	- `modules/`
	- `.west/` metadata

- Project-owned (safe to edit for application work):
	- `esp32_led_blink/app/`
	- `pico_led_blink/app/`
	- `native_project/app/`
	- `native_project_2/app/`
	- `scripts/` (if intentionally improving workspace automation)

Recommendation: prefer editing application and script folders, and avoid ad-hoc edits in vendored upstream trees unless you intentionally maintain a Zephyr fork workflow.

## 9. Troubleshooting Matrix

| Problem | Likely cause | Fix |
|---|---|---|
| `west build` fails with missing SDK for ESP32/Pico | SDK path missing | Ensure `zephyr-sdk-0.16.8/` exists and rerun `sh scripts/setup.sh <project>` |
| Native build toolchain mismatch | Host toolchain flag missing | Rebuild with `-- -DZEPHYR_TOOLCHAIN_VARIANT=host` |
| ESP32 flashing fails | Wrong serial device | Check `/dev/ttyUSB0` vs `/dev/ttyACM0` and adjust `--port` |
| Pico app compile error about `led0` | Board or DTS alias mismatch | Use `rpi_pico` board or ensure board DTS defines `led0` alias |
| Unexpected west behavior after project switch | Active manifest changed | Check current manifest via `west config manifest.path` |
| `esp32_env.sh` not found | Installer not run yet | Run `sh scripts/install_esp32_toolchain.sh` to generate it |

## 10. Validation Checklist

1. Run setup for each project once:

```sh
sh scripts/setup.sh esp32_led_blink
sh scripts/setup.sh pico_led_blink
sh scripts/setup.sh native_project
sh scripts/setup.sh native_project_2
```

2. Verify each expected artifact exists under each project's build directory.
3. Run native binaries and verify printed strings.
4. Flash ESP32/Pico and verify LED behavior.
5. Confirm active manifest before troubleshooting build issues:

```sh
west config manifest.path
```
